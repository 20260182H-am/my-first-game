## AI 대화 기록
01. 파티클에 콜리전을 부여하는 방법은?

AI 답변: 파티클의 위치가 화면의 좌표 범위를 넘는지 확인하는 조건문을 작성하고,
  경계를 넘었을 경우 속도 값을 반대로 바꾸면 충돌 효과를 구현할 수 있다.

python
if self.x <= 0 or self.x >= WIDTH:
    self.vx *= -1

if self.y <= 0 or self.y >= HEIGHT:
    self.vy *= -0.8

적용 결과: 콜리전 기능 구현 성공

02. 파티클이 자연스럽게 사라지는 방법은?

AI 답변: alpha(투명도) 값을 추가하고 매 프레임마다 감소시키면
파티클이 점점 투명해지면서 자연스럽게 사라지는 효과를 만들 수 있다.

self.alpha = 255
self.alpha -= 2

적용 결과: 갑자기 사라지지 않고 점점 흐려지며 자연스럽게 사라지는 효과 나타남

03. 파티클의 라이프타이밍을 늘리는 방법은?

AI 답변: 파티클의 life 감소 속도를 줄이거나 alpha 감소 속도를 줄이면
파티클이 화면에 더 오래 유지되도록 만들 수 있다.

self.life -= 0.5
self.alpha -= 2

적용 효과: 기존보다 파티클이 오래 유지되며 collision 효과도 더욱 명확히 확인이 가능

## 어려웠던 점(시행착오 및 해결 과정)

처음 Git Bash와 GitHub를 사용하다 보니 여러 가지 오류가 발생했다.
특히 git add, git commit, git push 과정에서 명령어 입력에 오타가 발생해 작업이 정상적으로 진행되지 않는 경우가 있었다.

또한 vim 편집기를 수정하거나 빠져나오는 방법을 몰라 AI의 도움을 구한 적도 있었고, git add 명령어에서 파일을 지정하지 않아
정상적으로 commit과 push가 진행되지 않는 문제도 경험하였다.

# 콜리전이 되기 전에 파티클이 사라지는 문제에 대한 질문

콜리전 기능을 적용한 뒤 파티클이 자연스럽게 사라지도록 Alpha 감소 효과를 추가했는데,
대부분의 파티클이 충돌하기 전에 먼저 사라지는 문제가 발생했다.

이를 해결하기 위해 AI와 질의응답을 통해 여러 방법을 시도하였다.
예를 들어 콜리전 이후에 Alpha 감소가 시작되도록 하는 방법,
파티클의 라이프 타이밍을 더 길게 설정하는 방법,
그리고 Alpha 대신 파티클의 크기가 점점 줄어들도록 하는 방법 등을 실험해 보았다.

여러 방법을 적용해 본 결과 파티클의 라이프 타이밍을 늘리는 방식이
내가 의도한 파티클 효과와 가장 유사한 결과를 보여주었다.
이 과정이 “파티클의 라이프 타이밍을 늘리는 방법은?” 이라는 질문을
AI에게 추가로 하게 되는 계기가 되었다.

## 배운 점

Git Bash를 사용하면서 대부분의 오류가 명령어 입력 과정에서의
