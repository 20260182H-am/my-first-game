## AI 대화 기록
1. 교수님의 pygame 코드를 Thonny에 복사했는데 IndentationError가 발생하는 이유는 무엇인가?

AI 답변: 파이썬은 중괄호 대신 들여쓰기로 코드 블록을 구분하기 때문에, while문이나 for문 아래에는 반드시 동일한 깊이의 들여쓰기가 필요하다. 들여쓰기가 없거나 Tab과 Space가 섞이면 오류가 발생한다.

while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

적용 결과: 들여쓰기를 수정한 후 오류가 발생하지 않고 정상적으로 실행되었다.

질문: 코드에 x += 10을 추가했는데 원이 움직이지 않고 오류가 발생하는 이유는 무엇인가?

AI 답변: x 변수가 미리 정의되지 않은 상태에서 사용되었기 때문에 NameError가 발생한다. 변수를 사용하기 전에 반드시 초기값을 설정해야 한다.

x = 0
x += 10

적용 결과: x 값을 미리 정의한 후에는 오류 없이 정상적으로 원이 이동하였다.

질문: pygame.draw.circle(screen, BLUE, (x, y), 50, 30)으로 변경했는데 타원이 아니라 가운데가 뚫린 원이 나오는 이유는 무엇인가?

AI 답변: circle 함수의 마지막 값은 타원의 세로값이 아니라 선의 두께를 의미한다. 따라서 해당 코드는 두께가 있는 원(도넛 형태)을 그리게 된다. 타원을 그리고 싶다면 ellipse 함수를 사용해야 한다.

pygame.draw.circle(screen, BLUE, (x, y), 50, 30)  # 도넛 형태
pygame.draw.ellipse(screen, BLUE, (x, y, 100, 60))  # 타원


적용 결과: circle 함수는 도넛 형태로 출력되었고, ellipse 함수를 사용했을 때 원하는 타원 형태가 출력되었다.

## 어려웠던 점 (시행착오 및 해결 과정)

파이썬의 코드는 직관이 아니라 논리를 기반으로 작성된다는 것을 다시 한 번 느끼게 되었다.
특히 x += 10이나 pygame.draw.circle(screen, BLUE, (x, y), 50, 30)과 같은 코드를 작성할 때, 내가 생각한 결과와 실제 결과가 다르게 나타났는데 이는 코드의 구조와 의미를 정확히 이해하지 못했기 때문이었다.

이 경험을 통해 파이썬이 비교적 쉬운 언어라고 하더라도 모든 코드는 명확한 규칙과 논리에 따라 동작한다는 것을 알게 되었고, 단순히 감각적으로 코드를 작성하는 것이 아니라 구조를 이해하며 접근해야 한다는 것을 깨달았다.

또한 문제가 발생했을 때 AI를 활용하여 원인을 분석하고 해결하는 과정에서, AI를 단순한 도구가 아니라 학습을 위한 보조 수단으로 활용하는 태도의 중요성도 느끼게 되었다.

## 배운 점
파이썬은 들여쓰기에 매우 민감한 언어이며, 들여쓰기가 곧 코드의 구조를 의미한다는 것을 알게 되었다.
특히 while문이나 for문 이후에 들여쓰기가 제대로 이루어지지 않으면 오류가 발생하며, 들여쓰기의 깊이가 일정하지 않아도 문제가 발생했다.

또한 변수는 반드시 정의된 이후에 사용해야 한다는 것 역시 알게 되었다.

이번 과정을 통해 코드 작성 시 논리적인 사고가 중요하다는 점과, 문제 해결 과정에서 AI를 적극적으로 활용하는 것이 학습에 큰 도움이 된다는 것을 깨달았다.
